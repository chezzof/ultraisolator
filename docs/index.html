Safety is a product surface

Strong controls. Clear escape routes.

UltraIsolator changes sensitive Windows scheduling controls, so it treats protection and recovery as first-class behavior.

Protected by default

Windows system processes, anti-cheat-sensitive paths, Steam, FACEIT, and terminal hosts stay outside background jailing.

Opt-in where it matters

Background jailing starts disabled. Conservative anti-cheat mode is available for stricter environments.

Recovery survives the session

Power-plan and process state are captured for restoration, including recovery after an interrupted run.

Local and inspectable

The app runs locally, does not download executable code at runtime, and publishes its implementation under the MIT license.

Before you install

Questions worth answering.

Does UltraIsolator guarantee more FPS?

No. Its goal is to reduce avoidable background scheduling noise and make tuning state visible. Performance varies by game, hardware, software, and workload.

Why does it need Administrator access?

Full CPU Sets, process priority, power-plan, timer, and recovery behavior use Windows controls that require elevated permissions.

Is background jailing enabled automatically?

No. It is opt-in and disabled by default. Review the configuration and protected-process list before enabling it.

What happens when the game closes?

After the configured exit delay, UltraIsolator restores tracked process placement, priority, timer state, and the original power plan.

Can I inspect the code and benchmark data?

Yes. The source, tests, build instructions, security notes, and structured CS2 VProf summary are published in the GitHub repository.
